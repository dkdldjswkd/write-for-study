인덱스 : 레코드를 빠르게 찾을 수 있도록 보조파일 (화일에 대한 또 다른 접근 경로)
-> 각각의 데이터 블럭의 주소를 가지고 있음, 책의 색인같은 개념(레코드 주소를 넣으면 인덱스 파일 크기가 커짐)

인덱스엔트리 : 인덱스 화일의 레코드

인덱스의 구조 : 키값, 데이터 블럭의 주소

인덱스 파일에서의 삽입
->인덱스 엔트리내에서 순차검색, 빈공간이 있는지 확인 후 없다면 반반 나누어 새로운 데이터 블럭을 만듬,
레코드 삽입, 레코드의 변화를 확인 후 인덱스 엔트리 레코드 변경

인덱스 파일에서의 삭제
-> 인덱스 엔트리 내에서 순차검색, 삭제하고 50% 이하면 합침, 레코드 변화시 인덱스 엔트리 레코드 변경
