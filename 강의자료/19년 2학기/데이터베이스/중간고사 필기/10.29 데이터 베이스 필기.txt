-> 운영과 목적에 맞는 데이터 수집이 필요함
2. 모든 주요 응용과 사용자들이 요구하는 데이터, 데이터 간의 관계를 표현하는 것

데이터 베이스 설계의 요구사항
1. 데이터의 모든 측면을 표현
2. 중복 최소화 : 데이터 불일치 -> 신뢰성이 떨어짐★★
3. 효율적 접근 : 성능에 영향을 미칠 수 있음
4. 데이터 무결성 : 데이터가 마땅히 지켜야 하는 조건(나이가 음수?)
5. 일관성 및 이해 용이 : 이해하기 쉬워야한다

ER 모델 ( Entity Relation Model ) (엔티티는 직사각형, 에트리 뷰트는 타원으로 표시)
실세계를 엔티티, 애트리뷰트, 엔티티들 간의 관계로 표현함
강한 표현력, 고도의 추상화, 이해 용이
엔티티 : 독립적으로 존재하면서 고유하게 식별이 가능한 객체 (학생, 학과, 남자, 여자, 명사나 명사형으로 표현됨) 
-> 엔티티 타입과 인티티 집합?
애트리뷰트 : 엔티티를 구성하는 요소 (이름, 학번) 엔티티가 레코드면 애트리뷰트는 필드
-> 도메인 : 애트리 뷰트가 가질수있는 값들의 집합(성별은 남, 여)
-> 단순 애트리뷰트 : 더이상 다른 애트리 뷰트로 나눌 수 없는 애트리뷰트
-> 복합 애트리뷰트 : 주소라는 애트리뷰트는 시, 동 으로 나누어 질 수 있음 (단순, 복합으로 나누는건 모델링하는사람 마음, 에트리뷰트에 다른 에트리뷰트가 이어짐)
-> 단일값 애트리뷰트 : 하나의 값을 갖는 애트리뷰트 (아이디, 이름)
-> 다치 애트리뷰트 : 여러개의 값을 가질 수 있는 애트리뷰트 (취미, 타원두개)
-> 저장된 애트리뷰트 : (급여)
-> 유도된 애트리뷰트 : 다른 애트리뷰트 값으로부터 얻어진 애트리뷰트 (엑셀에서의 변화값?, 점선의 타원)
-> 키 애트리뷰트 : 엔티티 타입에서 각 엔티티가 유일한 값을 가지는 애트리뷰트
(동일한 값이 존재하면 안된다, 최소한 하나의 키 애트리뷰트가 존재해야한다, 학생 엔티티의 학번, 여러개 있을 수 있다, 밑줄을 긋는다, 밑줄그은게 모두 합쳐진게 키 에트리뷰트)
관계 : 두개 이상의 엔티티들을 연관짓는 것 (수강, 결혼, 썸, 동사나 서술어로 표현됨)
