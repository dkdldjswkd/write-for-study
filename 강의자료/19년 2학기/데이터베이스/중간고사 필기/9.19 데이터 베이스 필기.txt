데이터 베이스의 특징 :

데이터 베이스 시스템의 자기기술성 - 메타-데이터
데이터를 "직접" 알고있지 않아도 액세스 할 수 있다.
메타 데이터를 통하여 간접적으로 엑세스함, 데이터 저장구조가 변경되어도 프로그램은 영향을 받지않음

프로그램과 데이터의 분리 - 의존성 해결

데이터의 추상화 - 모델링
저장구조와는 별도로 데이터의 의미를 표현하는 방법을 제공한다.
데이터의 구조가 모델과는 자유로워진다

데이터에 대한 다양한 뷰
사용자는 관심있는 데이터베이스의 일부를 볼 수 있음 (보는 사람마다 제공되는 정보가 조금씩 다름)

여러 사용자의 동시 사용 - 동시성 제어

데이터베이스의 스키마, 상태
스키마 : 데이터 베이스의 구조 (데이터가 들어가는 틀, 필드구조), 대단히 신중하게 만들어야함
데이터 베이스의 상태 : 데이터 베이스의 내용(데이터의 값)

데이터 베이스의 기능:
데이터 중복의 최소화
동시성 제어
보안기능
다양한 사용자 인터페이스
데이터 사이에 존제하는 복잡한 관련성을 표현
데이터 베이스의 무결성을 보장(신뢰성)
백업과 복구 기능

DBMS의 사용효과
표준화된 데이터 관리
데이터 구조 변경에 융통성 부여
개발시간 단축
항상 최신의 정보 제공
규모의 경제성 (데이터 관리비용)


데이터 베이스의 단점:ㄴ
비용증가 - 규모가 작은 회사에서는 손해볼 수 있음
성능감소 - 큰 규모에 사용하면 성능이 좋지만 적은 규모에서 사용하면 엑셀이 더 빠를 수 있음
고장영향의 확대 - 한곳에 문제가 생기면 문제가 커질 수 있다
